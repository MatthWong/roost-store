# Personas

## Persona 1: Student Customer
- Role: Primary buyer
- Goals:
  - Browse food and merchandise quickly
  - Place pickup-friendly orders during school schedule windows
  - Check order readiness with minimal friction
- Frustrations:
  - Unclear pickup times
  - Out-of-stock surprises
  - Complex checkout experience
- Key needs:
  - Mobile-friendly browsing
  - Clear product, price, and pickup details
  - Reliable order status lookup with order number + receipt code

## Persona 2: Teacher Customer
- Role: Secondary buyer with predictable scheduling needs
- Goals:
  - Place orders efficiently between classes
  - Plan pickup based on open store windows
  - Submit custom staff/team requests when needed
- Frustrations:
  - Ambiguous custom-order timelines
  - Inconsistent status communication
- Key needs:
  - Straightforward checkout via Square Payment Links
  - Clear custom-order constraints and expected turnaround
  - Accurate readiness status page

## Persona 3: DECA Member (Operations - Limited View)
- Role: Active club member handling day-to-day fulfillment support
- Goals:
  - View order queue essentials
  - Validate pickups using order number and receipt code
  - Maintain smooth handoff during open hours
- Frustrations:
  - Missing or inconsistent order identifiers
  - Access to too much sensitive data or too little operational detail
- Key needs:
  - Limited operations view (order number, order summary count, status)
  - Reliable status update workflow under sponsor-defined permissions

## Persona 4: DECA Officer (Operations - Full View)
- Role: Student leader responsible for operations execution
- Goals:
  - Monitor full order pipeline
  - Update status correctly and on time
  - Coordinate custom-order handling and pickup readiness
- Frustrations:
  - Delayed or manual reconciliation across forms and sheets
- Key needs:
  - Full order visibility in Google Sheets operations views
  - Controlled edit permissions
  - Clear process for fallback mode vs Square API mode

## Persona 5: Sponsor/Teacher Admin
- Role: System owner and policy gatekeeper
- Goals:
  - Manage club roster and permissions
  - Control catalog updates and new product promotion
  - Keep workflow compliant with school constraints (Google Sites/Forms/Sheets only)
- Frustrations:
  - Risk of accidental data exposure
  - Fragile handoffs during high-volume periods
- Key needs:
  - Administrative control over member access and data visibility
  - Maintainable process for adding products/services
  - Safe future switch to Square API when credentials are approved
